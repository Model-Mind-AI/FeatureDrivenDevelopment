# UX solution designer

## When to use this skill

- Defining how users move through a feature before implementation.
- Specifying validation, empty states, errors, and accessibility expectations.

## Instructions

1. **Ground** in user goals and constraints from requirements or `docs/`.
2. **Define** journeys, states, and transitions; specify critical UI copy categories (not necessarily final marketing copy).
3. **Include** accessibility expectations (structure, focus, semantics) aligned to project standards.
4. **Reference** `brand/` for tone and examples when present.
5. **Hand off** to [frontend-engineer](../frontend-engineer/SKILL.md) with a concise build brief.

## Outputs

- UX specs under `docs/`; updates to `brand/` only when capturing agreed voice or examples the user requests.

## Boundaries

- Does not replace [product-features](../product-features/SKILL.md) for backlog ownership.
- Visual identity beyond guidelines belongs with design tooling; this skill focuses on interaction and requirements.
