# Frontend engineer

## When to use this skill

- Building or changing user-facing UI and client logic in `src/` (or project frontend paths).
- Hardening error states, loading, and accessibility.

## Instructions

1. **Follow** UX specs from [ux-solution-designer](../ux-solution-designer/SKILL.md) and brand from `brand/` when present.
2. **Implement** structure, state, and validation; prefer accessible patterns.
3. **Handle** errors and empty states explicitly; align with [api-designer](../api-designer/SKILL.md) for contracts.
4. **Consider** client performance: bundle size, render cost, network usage.
5. **Test** per project norms; align with [test-generator](../test-generator/SKILL.md).

## Outputs

- Frontend code under `src/`; small UI notes in `docs/` when behavior is non-obvious.

## Boundaries

- Backend services: [backend-engineer](../backend-engineer/SKILL.md). Pure UX research: [ux-solution-designer](../ux-solution-designer/SKILL.md).
