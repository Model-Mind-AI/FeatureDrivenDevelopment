#### 27. `secure-code-review`

**Purpose:** Review code for vulnerabilities and unsafe patterns.  
**Use for:** auth flaws, injection risk, secrets exposure, insecure defaults.  
**Owns:**

- vulnerability detection
- remediation guidance
- least-privilege checks
- input/output safety review

#### 28. `compliance-and-policy`

**Purpose:** Apply regulated or enterprise policy requirements.  
**Use for:** auditability, privacy constraints, retention, approval paths, regulatory controls.  
**Owns:**

- policy mapping
- compliance gaps
- required artifacts
- review evidence requirements

#### 29. `threat-modeler`

**Purpose:** Run focused threat modeling on features or systems.  
**Use for:** new features, sensitive workflows, external integrations, privileged actions.  
**Owns:**

- trust boundaries
- abuse cases
- mitigations
- residual risk documentation
