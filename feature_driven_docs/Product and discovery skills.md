#### 4. `product-features`

**Purpose:** Create, revise, expand, critique, and normalize features.  
**Use for:** features, epics, user stories, acceptance criteria, non-functional requirements.  
**Operations inside one skill:**

- create
- edit
- decompose
- clarify
- gap-check
- acceptance-criteria generation

#### 5. `business-process-designer`

**Purpose:** Model workflows, actors, decisions, and exceptions.  
**Use for:** process mapping, workflow automation design, future-state process design.  
**Owns:**

- business flows
- edge cases
- approvals
- escalation logic
- swimlane style breakdowns

#### 6. `ux-solution-designer`

**Purpose:** Turn requirements into UX flows, screens, interaction models, and usability constraints.  
**Use for:** screen definitions, UI behavior, user flow design, validation logic.  
**Owns:**

- user journeys
- state transitions
- screen-level requirements
- form behavior
- accessibility expectations

#### 7. `backlog-and-sprint-planner`

**Purpose:** Convert requirements into executable work.  
**Use for:** sprint plans, story breakdown, sequencing, dependency mapping.  
**Owns:**

- backlog structure
- story slicing
- technical dependencies
- MVP vs later phase scoping
